# Multi-Module HubSpot Architecture & Governance

## Core Principle
Companies = customers  
Deals = module-level opportunities

Never use one deal for multiple modules.

## Pipelines
Each module has its own pipeline.

## Global Properties
- Product/Module
- Deal Type
- Deal Health
- Decision Outcome

## Workflow Governance
Workflows must be pipeline-scoped.

## Reporting
Use company dashboards and cross-pipeline reports. No parent deals.

## Enterprise Handling
- Enterprise is a deal type, not a pipeline
- Separate deal per module
- Linked to same company
- Optional grouping ID

## Not Allowed
- Parent-child deals
- Single enterprise pipeline
- Global blocking workflows

## Change Control
Leadership approval required.
