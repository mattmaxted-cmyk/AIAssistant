## Use Cases — Primary Focus

### What a use case page is

A use case page targets a specific workflow or problem a construction professional searches for. It is distinct from a module/Solutions page (which describes a product) and from a blog post (which is educational). A use case page is:

- Built around a single primary keyword that matches how the target persona searches
- 800–1,200 words
- Structured to convert: problem → how StoneRise solves it → features → FAQ → demo CTA
- The internal link target from supporting blog posts, concentrating topical authority on the core keyword page

### Page structure template

Every use case page follows this structure:

1. **H1** — contains the primary keyword exactly
2. **The Problem** — construction-specific context and consequences (legal, financial, operational)
3. **How StoneRise Solves It** — specific features, not generic benefits
4. **Key Features** — bullet list of relevant capabilities
5. **FAQ** — 4–6 questions targeting long-tail variants (FAQPage schema markup)
6. **CTA** — Book a Demo with UTM parameters

### UTM format for use case pages

```
https://www.stonerise.tech/use-cases/[slug]?utm_source=blog&utm_medium=organic_search&utm_campaign=[cluster]_commercial&utm_content=inline_link
```

## Content Rules

These apply to all blog posts and use case pages.

- No em dashes (—). Use colons, commas, or separate sentences instead
- Target keyword must appear in: H1, meta description, at least one H2, first 100 words
- 2–4 internal links per page (at least one in the first third; CTA counts as one)
- 1–3 external links to authoritative sources (RICS, CIOB, gov.uk, HSE, industry research)
- FAQPage schema markup on all use case pages and recommended on blog posts
- CTA must include UTM parameters (see format above)
- Do not call StoneRise an "ERP", "platform", or "suite" in blog copy
- Do not invent statistics — approved stats only: 3 min invoice processing, 20% time saved per order, 90% fewer admin errors, 75% faster approvals, 90% fewer delivery disputes
- Minimum word count: 800 words (blog), 800–1,200 words (use case pages)
