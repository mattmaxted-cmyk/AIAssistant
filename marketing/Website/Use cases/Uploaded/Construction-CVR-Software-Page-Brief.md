### 2. Problem Block — 3 Pain Points
Present as short, punchy statements. Each should feel like something the reader recognises from their own day:

- Building a CVR in Excel requires pulling figures from multiple sources — subcontract orders, valuations, invoices, variations — and reconciling them by hand. It takes days of commercial time each month just to produce a snapshot that is immediately out of date
- When the CVR is built on stale data, margin erosion is invisible until the month-end review. By that point the money has already moved and corrective action is weeks behind where it needs to be
- Inconsistencies between one QS's CVR format and another's make it impossible for directors to compare project performance across the business. Every report looks different and each one requires explanation before any decision can be made

### 3. Solution Block — How Certify Works
Four features, each as a heading plus one sentence of benefit:

- Live CVR generation — Certify produces a cost value reconciliation from live contract data at any point, not just at month end, so the commercial team always has a current picture of cost against value
- Automated cost capture — committed costs from subcontract orders, approved variations and payment applications feed into the CVR automatically, removing the manual data gathering that makes monthly reporting so time-consuming
- Margin and risk visibility — each CVR shows current margin, forecast final margin and any uncommitted costs or unrecovered variations that represent a risk to the final account
- Consistent format across all projects — every CVR is produced in the same structure regardless of which QS is running the project, giving directors a comparable view of the whole portfolio in one place
